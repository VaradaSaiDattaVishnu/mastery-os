This track is a gym, not a library. The unit of learning is a **solved problem**, not a read page. Every pattern card below is ≤1 minute of reading; everything else is you typing code into the Dojo until the pattern lives in your hands.

## The nightly protocol (run this loop)

1. **Pick the next problem** in the ladder. Read it. Start a 15-minute timer.
2. **Fight bare-handed.** No hints, no solution. Being stuck IS the workout — retrieval under strain is what builds the neural path. Talk out loud while you try (you're also training the interview voice).
3. **Stuck at 15?** Read ONLY the pattern card's template. Back to the code. 10 more minutes.
4. **Still stuck?** Read the solution like a rival's chess move — then **close it and rewrite from a blank editor**. Copying teaches nothing; reconstruction teaches everything.
5. **Green tests = say the script.** Out loud, 30 seconds: *"Pattern is X because [cue]. Time O(...), space O(...). Edge cases: [...]"* — FAANG grades the narration as much as the code.
6. **Mark for re-solve.** Anything that needed step 3+ goes on tomorrow's list. A problem is *owned* only when you've solved it cold, from blank, twice, days apart.

## The math of this track

~50 problems × the protocol ≈ 3–4 weeks of nights. That covers the patterns behind the vast majority of FAANG coding rounds. Volume with reconstruction beats coverage with reading — the JEE rule: the kid who solved 500 problems beats the kid who read 5 books, every time.

## Rules of the gym

- Never read two pattern cards in a row. Read one → solve its set → only then move.
- Re-solves before new problems, always. (Spaced repetition is the only cheat code that's real.)
- If a problem takes <5 minutes, you're reviewing, not training — skip forward.
- Log every solve in your head with one line: *cue → pattern → complexity*. That's the whole flashcard.
